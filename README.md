# 甲醛数据 Dashboard

一个用于展示 Supabase 中 `public.ch2o` 甲醛传感器数据的 Next.js Dashboard，可直接部署到 Vercel。

## 数据表

默认读取以下表结构：

```sql
create table public.ch2o (
  id bigint generated by default as identity not null,
  ppm_value numeric null,
  ppm_ct timestamp with time zone null,
  create_time timestamp with time zone null default now(),
  constraint ch2o_pkey primary key (id)
);
```

页面会优先使用 `ppm_ct` 作为记录时间；如果为空，则回退到 `create_time`。

## 本地运行

```bash
cp .env.example .env.local
corepack pnpm install
corepack pnpm dev
```

然后打开 `http://localhost:3000`。

## 环境变量

必填：

```bash
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-supabase-anon-key
```

可选：

```bash
FORMALDEHYDE_TABLE=ch2o
FORMALDEHYDE_TIMESTAMP_COLUMN=ppm_ct
FORMALDEHYDE_FALLBACK_TIMESTAMP_COLUMN=create_time
FORMALDEHYDE_VALUE_COLUMN=ppm_value
FORMALDEHYDE_LIMIT=120
```

## Supabase 权限

如果使用 `anon` key，确保 `ch2o` 表允许匿名读取。可在 Supabase SQL Editor 中按需启用 RLS 策略：

```sql
alter table public.ch2o enable row level security;

create policy "Allow public read ch2o"
on public.ch2o
for select
to anon
using (true);
```

如果你不希望公开读权限，也可以把读取逻辑改成使用服务端专用 key，并只在 Vercel 环境变量中保存。

## 部署到 Vercel

1. 将代码推送到 GitHub。
2. 在 Vercel 导入该仓库。
3. 在 Project Settings → Environment Variables 添加 `SUPABASE_URL` 和 `SUPABASE_ANON_KEY`。
4. 部署即可。

默认 Build Command 为 `corepack pnpm build`，Output Directory 保持 Next.js 默认值。
